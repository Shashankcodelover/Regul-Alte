=== Slide 7 ===
Proposed System / Methodology
Tattvasphere
ScanSkip HackOS-ONE
https://hackos.scanskip.com/
How It Works — 4 Steps
Step-by-step flow (show as numbered pipeline):
->
Step 1
—
INGEST
📥
User uploads a PDF. PyMuPDF + Tesseract OCR extracts every clause and assigns it a unique ID with page number. Output: structured JSON clause list.
->
Step 2
—
RED-TEAM
⚔️
The Attacker Agent reads every clause and attempts to find exploitable loopholes — like a hostile lawyer. The Defender Agent argues back. Any clause the Attacker wins = flagged dangerous.
->
Step 3
—
VALIDATE
🔬
The Z3 SMT Solver maps all contract rules as logical equations and checks for contradictions — clauses that cancel each other out or create rights that can never actually be used.
->
Step 4
—
OUTPUT
📤
Risk Score (0–100) generated. AutoFixer rewrites dangerous clauses. Obligation timeline shows every deadline. Redlined .docx exported.

The 5 Core Features
Feature cards (one per row or 2x3 grid):
🔍 Loophole Finder —
Two AI agents debate every clause. Attacker hunts exploits, Defender assesses real risk. You see exactly which clauses a vendor lawyer could use against you.
⚖️ Logic Validator —
Z3 solver reads the contract like a math problem. Finds clauses that contradict each other or create "paper rights" that can never be exercised.
🚨 Danger Risk Score —
Every clause gets a 0–100 danger score. The whole contract gets an overall score. Red = dangerous. Green = safe. No law degree needed to understand it.
📌 Anti-Hallucination Tracker —
Every AI claim links back to the exact sentence and page number in the original PDF. One click verifies any finding.
✏️ Auto-Fixer + Redline Export —
Dangerous clauses are rewritten into fair versions automatically. One button exports a professional .docx with original text struck through in red and fix in green — exactly like a real lawyer's markup.
